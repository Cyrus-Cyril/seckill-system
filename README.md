# 商品库存与秒杀系统

## 项目简介

本项目是一个面向电商场景的商品库存与秒杀系统课程作业，目标是完成一个具备基础业务能力和一定高并发设计思想的后端系统原型。系统围绕用户、商品、库存、订单四个核心业务对象展开，重点体现秒杀业务下的库存控制、下单流程和系统拆分思路。

本项目当前采用“设计上按微服务拆分，开发上先使用单体模块化实现”的方式推进。这样既能够展示完整的系统设计能力，也便于在课程作业阶段快速完成开发、调试与演示。

## 项目目标

本项目主要实现以下内容：

1. 完成商品库存与秒杀系统的整体设计
2. 绘制系统架构图、数据库 ER 图
3. 定义核心服务的 RESTful API
4. 完成基础开发环境搭建
5. 初始化项目代码仓库并建立规范目录结构
6. 实现用户注册与登录功能
7. 为后续商品管理、库存管理、订单处理与秒杀功能预留扩展空间

## 系统功能概述

系统计划包含以下核心模块：

### 1. 用户模块
负责用户注册、登录、用户信息查询等功能。

### 2. 商品模块
负责商品新增、修改、上下架、详情查询、列表展示等功能。

### 3. 库存模块
负责库存查询、库存扣减、库存回滚、库存锁定等功能，是秒杀业务的核心支撑模块。

### 4. 订单模块
负责普通下单、秒杀下单、订单查询、订单取消等功能。

## 项目特点

1. 采用分层设计，结构清晰，便于维护
2. 按用户服务、商品服务、库存服务、订单服务进行职责划分
3. 面向秒杀场景设计库存控制逻辑，降低超卖风险
4. 预留 Redis 缓存与消息队列扩展方案，具备后续优化空间
5. 适合作为课程作业、实验项目和后续扩展练习的基础工程
